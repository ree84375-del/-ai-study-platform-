AI 治理與最佳實踐指南

確保 Antigravity 平台穩定、合規且高效運作的終極準則

1. 能力協作 (Capability)

  • Skills (技能): 利用 Skill Creator 建立專屬技能（如 SEO 檢查、資料庫 CRUD）。避免盲目猜測。
  • MCP (模型上下文協議): 透過 MCP Server 串接資料庫或知識庫，減少幻覺，提高即時性。
  • 檔案引用: 在對話時使用 @檔案名 明確指出相關代碼。
  • Planning Mode: 複雜任務必須先產生「實作計畫」，經審核後再動工，提升效率並降低錯誤率。

2. 環境與穩定性 (Stability)

  • 環境配置: 定期檢查 Windows 路徑問題（HOME 變數）與終端機盲點。
  • 配額管理 (Quota): 若回應變慢或品質下降，應檢查是否觸發限額。視情況切換模型（如 Gemini -> Claude）。
  • 防濫用機制: 控制 API 調用頻率，避免產生大量 429/503 錯誤而被系統標記。

3. 安全性與合規性 (Compliance)

  • 拒絕包殼工具: 嚴禁使用 OpenClaw / OpenCode 等第三方插件，避免違反 ToS。
  • 憑證安全: 不共用 OAuth Token。
  • 人工審核: 將 Terminal/JS Execution 改為 "Request Review"。完全自動化執行易被標記為異常。

4. 行為準則 (Ethics)

  • 隱私保護: 不讓 Agent 存取與專案無關的個人敏感資料。建議環境隔離。
  • 合規內容: 嚴禁生成詐騙、釣魚、惡意軟體或侵權內容。
本指南應作為平台 AI 維護的最高準則。若有修補或更新，請對 Agent 下達同步指令。
